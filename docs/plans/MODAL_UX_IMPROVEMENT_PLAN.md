# 마이 설정 모달 UX 개선 작업 분해 명세서

## 1. 현재 마이 모달 UI 구성 분석
현재 프로젝트의 설정 모달은 크게 두 곳에서 관리되고 있습니다.
*   **`index.html`**: 메인 페이지 전용 모달. 계정, 메뉴, 배경(프리셋/단색), 이미지 주소/드롭존, Pro 멤버십, 백업, 초기화 등 모든 설정 기능을 포함하고 있습니다.
*   **`src/shared.js` (`buildGlobalMyModalHTML`)**: `admin.html`, `owner.html`, `community.html` 등 서브페이지에 공통 주입되는 모달. 테마 설정 등을 제외하고 '계정'과 '메뉴' 항목만 포함되어 있습니다.

**문제점 및 한계:**
*   섹션 헤더가 단순한 텍스트(`<label>` 태그)로만 되어 있어 위계가 약함.
*   메뉴 항목이 아이콘 없는 단순 텍스트 버튼/링크의 나열로 직관성이 떨어짐.
*   테마 이미지의 `alt` 값이 영어(Beach, Space 등)로 되어 있어 국내 사용자 친화력이 다소 부족.
*   배경이나 테마를 클릭해도 무엇이 선택되었는지 활성화(Active) 상태 피드백이 없음.
*   데이터를 덮어쓰는 '불러오기'나 '로그아웃' 시 사용자 실수를 방지하는 안전장치(Confirm)가 부족함.

---

## 2. 작업 단위 분해 및 우선순위 (구현 난이도 기준)

코드의 대규모 수정 없이 HTML/CSS 수준에서 즉각적으로 효과를 볼 수 있는 **'바로 구현 가능한 작은 변경'**과, JS 로직 추가가 필요한 **'후순위 변경'**으로 분리했습니다.

### 🟢 1단계: 바로 구현 가능한 작은 변경 (UI/텍스트 위주)

1.  **메뉴 아이콘 추가 (UI 향상)**
    *   **작업 내용**: 계정 메뉴 및 하위 메뉴(검색, 최근 방문, 내 트리, 커뮤니티 등) 텍스트 옆에 직관적인 16x16 또는 20x20 크기의 SVG 아이콘(Heroicons 등)을 배치하여 가독성 강화.
    *   **대상 파일**: `index.html`, `src/shared.js`

2.  **테마 이름 한국어화 (로컬라이징)**
    *   **작업 내용**: `index.html` 내 추천 배경 이미지 버튼의 `alt` 속성을 '해변', '우주', '자연', '도시', '노을', '추상' 등으로 변경. 단색 배경도 '기본', '화이트', '다크' 등으로 툴팁/aria-label 명확화.
    *   **대상 파일**: `index.html`

3.  **Pro 설명 구체화 (카피라이팅)**
    *   **작업 내용**: 한 줄로 된 막연한 설명을 명확한 불릿 리스트 혜택으로 변경.
        (예: `• 무제한 러브트리 생성`, `• 프리미엄 고화질 배경`, `• 모든 기기 동기화`)
    *   **대상 파일**: `index.html`

4.  **섹션 위계 강화 (UI 향상)**
    *   **작업 내용**: `계정`, `메뉴`, `추천 배경`, `백업` 등의 `<label>` 헤더에 섹션을 상징하는 작은 아이콘을 추가하고, 하단 여백(margin)을 미세 조정. 섹션 간의 구분을 더 명확히 하기 위해 옅은 구분선(divider) 활용.
    *   **대상 파일**: `index.html`, `src/shared.js`

### 🟡 2단계: 로직 추가가 필요한 후순위 변경 (JS 연동 위주)

1.  **선택 상태 표시 (상태 동기화)**
    *   **작업 내용**: 현재 적용된 배경(이미지 URL 또는 색상 코드) 상태를 읽어와, 모달이 열릴 때 해당하는 프리셋 버튼에 `ring-offset-2 ring-brand-500` 등의 활성 스타일을 추가하는 로직 구현.
    *   **대상 파일**: `index.html` (스타일 클래스 준비), `index.js` (모달 오픈 시점 로직)

2.  **파괴적 액션 확인 다이얼로그 (안전성 강화)**
    *   **작업 내용**: `로그아웃`, `백업 파일 불러오기`, `초기화` 액션 실행 직전에 브라우저 기본 `confirm()` 또는 커스텀 다이얼로그를 띄워 사용자의 재확인 요구.
    *   **대상 파일**: `index.html` (onclick 속성 변경), `index.js`, `src/auth.js` (실행 함수에 분기 추가)

3.  **드롭존 시각적/기능적 강화 (UX 향상)**
    *   **작업 내용**: 
        *   이미지 드래그 오버(`dragover`) 시 배경색이 변하는 피드백 추가.
        *   선택된 파일명이 있을 경우 드롭존 내 텍스트를 파일명으로 변경하여 표시.
    *   **대상 파일**: `index.html`, `index.js`

---

## 3. 예상 영향 범위 요약

*   **`index.html`**: 메인 모달의 마크업이 변경되며 CSS 클래스가 다수 추가됩니다. 다른 페이지의 레이아웃에는 영향을 주지 않습니다.
*   **`src/shared.js`**: 글로벌 모달 주입 함수(`buildGlobalMyModalHTML`)의 마크업이 변경됩니다. 이 함수는 전역으로 동작하므로 서브페이지(`community`, `owner` 등)의 UI도 함께 업데이트됩니다. `index.html`과의 통일성 유지가 중요합니다.
*   **`index.js` & `src/auth.js`**: 2단계 작업 진행 시 선택 상태 렌더링 함수와 확인(confirm) 프롬프트 로직이 추가됩니다. 기존의 동작 흐름(기능 자체)을 방해하지 않는 선에서 래핑(Wrapping) 형태로 추가해야 합니다.
