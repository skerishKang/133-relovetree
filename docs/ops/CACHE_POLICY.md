# 프로젝트 캐시/버전 운영 원칙

이 문서는 Relovetree 프로젝트의 정적 자산(CSS, JS, Image) 캐시 관리 및 버전 제어 규칙을 정의합니다. 모든 개발자 및 AI 에이전트는 이 규칙을 반드시 준수해야 합니다.

## 1. 기본 원칙
1. 이 프로젝트는 빌드 해시 시스템 대신 **query-string 버전 방식**으로 캐시를 관리한다.
2. 파일 내용이 실제로 바뀐 경우에만 해당 CSS/JS/이미지 참조에 `?v=YYYYMMDD_vN` 형식의 버전을 올린다.
3. 바뀌지 않은 파일의 버전은 불필요하게 올리지 않는다.

## 2. 버전 형식
- **형식**: `?v=YYYYMMDD_vN` (날짜 + 당일 리비전)
- **예시**: `?v=20260414_v4`
- 이미 적용된 버전 체계를 깨는 임의의 네이밍(`?cache=1`, `?ver=latest` 등)은 사용하지 않는다.

## 3. 공통 파일 관리 (Critical)
- 공통 파일(`shared.js`, `shared-layout.js`, `auth.js`, `app-base.css` 등)이 바뀌면, 그 파일을 참조하는 **모든 관련 페이지**에서 같은 새 버전으로 일괄 업데이트한다.
- 특정 페이지 HTML만 바뀌고 공통 CSS/JS가 안 바뀐 경우에는 공통 파일 버전을 올리지 않는다.

## 4. 진단 및 보고
- 캐시 문제를 의심할 때 무조건 전체 버전을 다 올리지 말고, 실제로 변경된 파일만 타겟팅하여 올린다.
- 작업 완료 후 보고서에는 반드시 다음 사항을 명시한다.
    - 실제 내용이 바뀐 파일 목록
    - 버전을 올린 참조 파일 및 변경된 버전 번호
    - 특정 파일만 버전을 올린 사유

## 5. 기타 자산
- 이미지, 아이콘 등 정적 자산이 교체된 경우에도 동일한 URL 쿼리 버전 스트링을 적용한다.
- 신규 파일을 직접 링크로 추가할 때도 현재 날짜 기준의 버전 규칙을 적용한다.
