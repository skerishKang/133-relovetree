# Lovetree User Flow

## Purpose

이 문서는 Lovetree의 핵심 사용자 흐름을 고정하기 위한 문서다.

디자인, 개발, QA는 모두 아래 흐름을 기준으로 판단한다.

## Primary Flow

가장 중요한 대표 흐름은 아래 하나다.

### Flow A: 첫 순간을 러브트리로 남기기

1. 사용자가 영상 링크를 발견한다
2. Lovetree로 가져온다
3. 특정 시점을 선택한다
4. 그 순간의 감정 메모를 남긴다
5. 새 트리에 넣거나 기존 순간 뒤에 연결한다
6. 저장된 러브트리를 다시 본다
7. 원하면 다른 팬에게 공유한다

이 흐름이 제품의 중심이다.

## Detailed Step Breakdown

### Step 1. Entry

사용자는 아래 진입점 중 하나로 시작한다.

- 링크 붙여넣기
- 모바일 공유하기
- 내 트리에서 새 순간 추가

### Step 2. Source Capture

사용자는 영상을 고르고 저장한다.

- source URL 확인
- 영상 썸네일 또는 미리보기 확인
- 필요 시 플랫폼 타입 확인

### Step 3. Moment Capture

사용자는 영상의 "왜 반했는지"를 남긴다.

- 특정 시점 선택
- 짧은 제목 작성
- 감정 태그 선택
- 메모 작성

여기서 메모는 선택이 아니라 핵심이다.

### Step 4. Connection

사용자는 이 순간을 러브트리 안 어디에 둘지 정한다.

- 첫 순간이면 루트로 저장
- 이미 트리가 있으면 기존 순간 뒤에 연결
- 모바일은 단순 연결
- 데스크톱은 구조를 더 넓게 조정 가능

### Step 5. Revisit

사용자는 저장 후 다시 감상한다.

- 트리 전체 보기
- 특정 메모리 상세 보기
- 이전/다음 순간 따라가기

### Step 6. Share

사용자는 원할 때 공유한다.

- 링크 공유
- 공개 범위 설정
- 읽기 전용 러브트리 감상
- 다른 팬의 공감 / 댓글 / 저장

## Supporting Flows

### Flow B: 내 러브트리 관리

1. 로그인
2. 내 러브트리 목록 확인
3. 최근 수정한 트리 열기
4. 공유 범위 조정
5. 필요하면 삭제 또는 복제

### Flow C: 다른 팬의 러브트리 감상

1. 커뮤니티 또는 링크로 진입
2. 트리 전체 흐름 감상
3. 공감 / 댓글 / 저장
4. 내 트리로 가져갈지 판단

### Flow D: 모바일 빠른 기록

1. 공유하기 또는 링크 붙여넣기
2. 순간 선택
3. 짧은 메모 작성
4. 저장
5. 나중에 데스크톱에서 더 정리

## Success Criteria

핵심 Flow A가 성공했다고 보려면 아래가 가능해야 한다.

1. 사용자가 영상 링크를 넣고
2. 특정 순간을 기록하고
3. 감정 메모를 남기고
4. 트리에 연결하고
5. 다시 열어도 그 흐름이 유지되고
6. 공유 링크로 다른 사람이 감상할 수 있어야 한다

이 여섯 조건이 만족되지 않으면 제품의 중심 경험이 아직 완성되지 않은 것이다.

## Failure Signs

아래 현상이 보이면 흐름이 깨진 것이다.

- 링크 저장은 되지만 감정 메모가 약하다
- 노드는 저장되지만 연결 이유가 약하다
- 모바일에서 연결 구조가 안 느껴진다
- 공유 화면이 영상 하나만 보여주고 트리 전체가 안 보인다
- 커뮤니티가 게시판처럼만 보인다

## Implementation Note

실제 개발 작업을 맡길 때는 "페이지 단위"보다 "흐름 단위"로 맡기는 것이 낫다.

예:

- "editor-desktop 구현"보다
- "영상 저장 → 메모 작성 → 트리 연결 → 다시 보기" 흐름 구현

이 방식이 Lovetree의 제품 방향을 덜 잃는다.
