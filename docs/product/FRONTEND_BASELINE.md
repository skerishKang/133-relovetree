# Lovetree Frontend Baseline

## 1. 개요 (Baseline Context)

`frontend-concept-v2` 시안은 Lovetree 제품의 시각적 목표이자 UX 가이드라인입니다. 이 문서는 단순한 시안 목록을 넘어, 실제 프로덕션 코드(`src/`, `pages/`)로 통합될 때의 기준을 정의합니다.

**중요 지침:**
- **시안은 가이드다**: 시안을 HTML로 복제하는 작업과 실제 기능을 연결하는 작업은 별개로 관리합니다.
- **제품 정체성 우선**: 단순한 영상 저장 앱처럼 보이지 않도록, 모든 화면은 "감정의 흐름"과 "트리 구조"를 강조해야 합니다.

## 2. 페이지 분류 및 구현 기준

### 2.1 핵심 페이지 (Flow A 전용 - P1)
사용자가 직접 기록을 남기고 감상하는 가장 중요한 화면들입니다.

| 페이지 | 파일명 | 상태 | 구현 가이드 |
|------|------|------|------------|
| **데스크톱 에디터** | `editor-desktop.html` | **추가 보완 필요** | 캔버스 좌표 시스템과 실제 DB 연동 로직의 통합이 핵심입니다. |
| **모바일 순간 추가** | `mobile-add-memory.html` | **기본 구성됨** | `src/mobile-add-memory.js`와 1:1 매핑. timestamp 필드 연동 필요. |
| **모바일 트리 감상** | `mobile-tree.html` | **시각화 보완 필요** | 단순 리스트에서 "연결선"이 보이는 형태로 고도화 필요. |
| **순간 상세 보기** | `memory-detail.html` | **기본 구성됨** | 플레이어와 메모 영역의 레이아웃을 실제 데이터와 연결합니다. |

### 2.2 서비스 보조 페이지 (P2)
진입점과 관리를 담당하는 페이지들입니다.

| 페이지 | 파일명 | 상태 | 구현 가이드 |
|------|------|------|------------|
| **랜딩 페이지** | `home.html` | **기존 index.html과 통합 필요** | 현재 `index.html`이 이미 랜딩으로 사용 중. 시안은 참고용. |
| **내 트리 목록** | `my-trees.html` | **기본 구성됨** | 트리 생성 및 관리 기능을 실제 `src/my-trees.js`와 연결합니다. |
| **로그인/마이페이지** | `login.html` | **기본 구성됨** | Firebase Auth 연동 및 로그인 유도 메시지 강화. |

### 2.3 커뮤니티 및 부가 기능 (P3~P4)
공유와 전파를 위한 페이지로, 나중에 본격적으로 다듬을 대상입니다.

| 페이지 | 파일명 | 상태 | 구현 가이드 |
|------|------|------|------------|
| **커뮤니티 메인** | `community.html` | **재설계 필요** | 단순 게시판 형태가 아닌 "트리 컬렉션" 중심으로 바궈야 합니다. |
| **공유 트리 상세** | `community-tree-detail.html` | **재설계 필요** | "한 영상"이 아닌 "한 팬의 사랑 경로 전체"를 보여줘야 합니다. |
| **설정 페이지** | `settings.html` | **우선순위 낮음** | 공개 범위(Visibility) 설정 등 필수 항목만 선구현. |

## 3. 현실적 구현을 위한 체크포인트

### 3.1 데스크톱 에디션 (Precision)
- **에디터 캔버스**: 시안의 `desktop-love-tree-board.png`는 고정된 크기처럼 보이지만, 실제로는 무한 캔버스나 노드 자동 배치가 지원되어야 합니다.
- **디테일 패널**: 우측 패널의 정보 위계(Title -> Date -> Tag -> Memo)를 실제 DB 필드와 일치시킵니다.

### 3.2 모바일 익스피리언스 (Flow)
- **추가 흐름**: `mobile-add-memory` -> `mobile-add-branch`로 이어지는 흐름이 실제 세션과 트리 ID를 잃지 않고 연결되어야 합니다.
- **트리 가독성**: `mobile-tree`에서 노드가 많아졌을 때의 스크롤 가독성을 고려해야 합니다.

## 4. 정체성 강화 포인트

- **연결의 시각화**: 모든 페이지에서 이전/다음 순간이 "연결되어 있다"는 시각적 단서(선, 화살표, 경로 표시 등)가 반드시 노출되어야 합니다.
- **감정의 강조**: 단순히 영상을 보는 것이 아니라, 그때 느낀 **감정 태그(설렘, 감동 등)**와 **사용자의 메모**가 영상만큼 큰 비중으로 보여야 합니다.

## 5. 단계별 실행 큐 (Build Queue 연동)

1.  **Phase 1**: 핵심 4페이지(Add Memory, Tree View, Detail, My Trees)의 UI를 시안 기준으로 프로퍼티를 마이그레이션.
2.  **Phase 2**: 에디터의 노드 카드 디자인을 시안의 고밀도 레이아웃으로 교체.
3.  **Phase 3**: 커뮤니티 페이지를 제품 정체성에 맞게 전면 재해석하여 구현.

## 9. 제품 정체성과 가장 잘 맞는 핵심 화면 3개

### 9.1 1위: editor-desktop

**정체성 적합도:** ★★★★★

**이유:**
- "사랑의 경로가 보이는 트리 구조" 핵심
- "데스크톱은 연결 구조와 편집 우선" 충족
- 노드가 가지처럼 연결된 시각화

### 9.2 2위: mobile-add-memory

**정체성 적합도:** ★★★★★

**이유:**
- "첫 순간을 쉽게 기록" 핵심
- "모바일은 빠른 기록 우선" 충족
- URL → timestamp → 감정 태그 → 메모 → 연결 흐름 자연스러움

### 9.3 3위: mobile-tree

**정체성 적합도:** ★★★★☆

**이유:**
- "기록을 다시 보기 쉽게" 핵심
- 세로 흐름으로 momento 연결 순서 표시
- "감정 흐름처럼 보여야 함" 충족
